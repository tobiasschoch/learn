Unit 1.2 Console


The console

The R statistical software is operated by the R console. The console is a command-line interface which executes the commands we type in. Only a very limited number of actions or commands can be chosen from the menu bar.



Wrap up

Clear the console

We can clear the console by pressing ctrl + L

Working with scripts

First, we select/highlight the line (or lines) of code we want to execute in the R Editor. Second, we use the shortcut ctrl + R (the "R" stands for "run") to execute or run the selected commands.

Alternative. Having highlighted the commands, we can execute them by right-clicking with the mouse on the highlighted code and then choosing "run".

Saving the workspace

The workspace is our current R working environment and it includes any data we generate or load. At the end of an R session, we can save the workspace (in addition to the R scripts) but this is not mandatory. Though, it can be useful to save the workspace if we have a lot data.

Working directory New

We haven't yet covered the notion of a "working directory". But now, we are going to address this. The notions "workspace" and "working directory" are quite often mixed up by beginning R users. Let's shed some light on it.

Any application or app on your computer has its "working directory". It is the location where the app "wants" to store the files. In R, this is where R will look for data and scripts you ask it to load. It is also where any files you write to disk will go.

  • Lisa's main computer is a Windows computer. On her main computer, the "working directory" is her "Documents" folder which is located on the disk at C:\Users\lisa\Documents (on the German edition of Windows, we have Benutzer and Dokumente in place of Users and Documents). Where is my "working directory"? Unless you haven't messed up your R installation, the working directory on your Windows computer will also be the "Documents" folder.
  • Lisa also has an Apple Mac computer. On her Apple Mac computer, the "working directory" is Lisa's home folder; her home folder has the name LISA. On your Apple Mac, you can use the shortcut ⇧ + ⌘ + H to get straight to the home folder with the finder (Note: the ⇧ key is the shift key not the arrow key).

How are the workspace and the working directory related? The working directory is the physical location on the disk where R wants to save the workspace to (and from where it loads the workspace). Experienced R users can move the working directory to another place on the disk, on another computer or into the cloud. However, we recommend to stick with the default location.


What more do you need to know?

Stop/ interrupt R New

We can stop the execution (any time) by pressing esc. This can be useful if "something" goes wrong, e.g. when we have a typo in our commands.


A line that starts with a plus ("+") New

Suppose we want to compute 2 + 3 * 4. However, in a hurry we type 2 + 3 * and mistakenly press enter; then, the console looks like this

 > 2 + 3 *
 +

This command is incomplete. The + indicates that R is expecting that we finish the command. We can either type 4 to complete our calculation and press enter or we stop the computation with esc (and re-do the calculation properly).


Error New

Errors happen all the time. For instance, in the following code snipped, we erroneously type x although x is not defined. Therefore, R tells us that there is no x. Note that the cursor (below the error message) is waiting for us to continue typing in commands.

 > x
 Error: Object 'x' not found
 >

Recall previous commands New

As you work with R you'll often want to re-execute a command which you previously entered. The R console supports the ability to recall previous commands using the arrow keys.

  • Up — Recall previous command(s) with the arrow key ↑
  • Down — Reverse of Up with the arrow key ↓

The Help system

R has a help system which describes the commands. We can access the help system directly from the console. Suppose we want to learn more on the command "median". To access the help system we type ?median. Then, R opens the help files on "median" in the web browser. So, all we have to do is put a ? (question mark) in front of the command we are interested in.
